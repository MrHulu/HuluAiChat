## 决策过程

### 原始方案 (被 Critic 修改)

| 方案 | 问题 | Critic 评分 |
|------|------|-------------|
| 多模型并行对比 | API 成本翻倍，用户投诉风险 | 需修改 |
| 代码片段收藏夹 | 复杂度高，需求不明确 | 需修改 |
| 会话内容搜索增强 | 现有功能已足够 | 取消 |
| 技术债务清理 | 用户无感知 | 延后 |

### Critic 关键发现

> **QuickPanel 持久化已经实现了！**
>
> 看 `QuickPanel.tsx` 第 117-127 行，每次打开 QuickPanel 都会创建一个**真实 session**。
> 这些对话已经保存在数据库里了！
>
> 真正的需求不是"持久化"，而是"QuickPanel 历史入口"。

### 最终决策

采纳 Critic 建议，v3.60.0 聚焦：
1. **QuickPanel 历史入口** - 让用户能找到之前的问题 (P0)
2. **剪贴板历史记录** - 记录最近处理的内容 (P1)
3. **侧边栏快速搜索** - 提升导航效率 (P2)

---

## MVP 功能范围

### Phase 1: QuickPanel 历史入口 (P0) - 1-2 Cycles

#### TASK-230: QuickPanel 历史入口

**背景**: QuickPanel 的对话已经持久化到数据库，但用户不知道去哪找。

**功能描述**:
- 侧边栏添加"快速对话"文件夹
- 该文件夹显示所有从 QuickPanel 创建的会话
- QuickPanel 关闭时有 Toast 提示"已保存到快速对话"
- 可以从快速对话"继续在主窗口讨论"

**验收标准**:
- [ ] 侧边栏显示"快速对话"文件夹
- [ ] QuickPanel 关闭时显示 Toast 提示
- [ ] 点击快速对话可在主窗口打开
- [ ] 自动生成摘要标题

**技术方案**:
- 给 sessions 表添加 `source` 字段 (`quickpanel` | `main`)
- 侧边栏按 `source` 分组显示
- Toast 组件复用现有的

**预计周期**: 1-2 Cycles

---

### Phase 2: 剪贴板历史记录 (P1) - 1-2 Cycles

#### TASK-231: 剪贴板历史记录

**背景**: 用户用 QuickPanel 处理了剪贴板内容，想找回之前处理的内容。

**功能描述**:
- 记录最近 50 条剪贴板处理历史
- QuickPanel 底部显示"历史"按钮
- 历史列表显示：原文摘要 + 处理类型 + 时间
- 点击历史项可重新使用

**验收标准**:
- [ ] QuickPanel 底部显示"历史"按钮
- [ ] 点击显示最近 50 条历史
- [ ] 每条历史显示原文摘要（前 50 字）
- [ ] 点击历史项自动填充到输入框

**隐私设计**:
- 历史记录仅存储在 localStorage
- 用户可清除历史
- 不上传到服务器

**预计周期**: 1-2 Cycles

---

### Phase 3: 侧边栏快速搜索 (P2) - 1 Cycle

#### TASK-232: 侧边栏会话快速搜索

**背景**: 用户的会话列表越来越长，需要快速找到目标会话。

**功能描述**:
- 侧边栏顶部添加搜索框
- 支持搜索会话标题
- 实时过滤显示匹配结果
- 键盘导航支持

**验收标准**:
- [ ] 侧边栏顶部显示搜索框
- [ ] 输入时实时过滤
- [ ] 支持键盘上下导航
- [ ] Esc 清空搜索

**预计周期**: 1 Cycle
