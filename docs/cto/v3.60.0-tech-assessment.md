## 1. 技术债务清单

### 1.1 Critical (P0) - 必须解决

| ID | 债务描述 | 位置 | 影响 | 状态 |
|----|---------|------|------|------|
| TD-001 | Rust 层未充分利用 | `src-tauri/src/lib.rs` | 性能潜力未释放 | OPEN |
| TD-002 | Sidecar 进程无自动重启 (Rust) | `lib.rs:165-170` | 后端崩溃无恢复 | BLOCKED (内存) |

**TD-002 详细分析**:

当前实现:
```rust
let (mut _rx, _child) = sidecar_command.spawn().expect("Failed to spawn sidecar");
```

问题:
- `.expect()` 会 panic，用户看到崩溃
- `_rx` 未使用，无法获取日志
- `_child` 未监控，崩溃后无法自动重启
- v3.59.0 通过前端健康监控绕过，但不是根本解决

**推荐方案** (需解决 Rust 编译内存问题):
1. 实现 `SidecarManager` 结构体管理进程生命周期
2. 监控 stdout/stderr 输出到日志
3. 指数退避自动重启 (最多 3 次)
4. 前端通过 Tauri command 获取状态

### 1.2 High (P1) - 本版本建议解决

| ID | 债务描述 | 位置 | 影响 | 优先级 |
|----|---------|------|------|--------|
| TD-003 | WebSocket 竞态条件 | `useChat.ts` | 会话切换时消息错乱 | HIGH |
| TD-004 | API 无请求超时 | `client.ts` | 网络差时挂起 | HIGH |
| TD-005 | 全局服务实例模式 | `openai_service.py` | 测试困难 | MEDIUM |
| TD-008 | API Client 文件过大 | `client.ts` (1500+ 行) | 维护困难 | MEDIUM |

**TD-003 竞态条件**:

问题场景:
1. 用户在会话 A 发送消息
2. 切换到会话 B，触发新 WebSocket
3. 会话 A 的响应到达，但当前 sessionId 已变为 B
4. 消息被添加到错误的会话

当前缓解措施: `currentSessionIdRef` 检查

**推荐方案**:
```typescript
// 为每个消息添加 correlation ID
interface SendMessageOptions {
  sessionId: string;
  correlationId: string; // UUID
  content: string;
}

// 响应中包含 correlationId，前端验证匹配
```

**TD-004 请求超时**:

当前代码:
```typescript
const response = await fetch(`${API_BASE}/health`);
// 无超时，可能永久挂起
```

推荐方案:
```typescript
async function fetchWithTimeout(
  url: string,
  options: RequestInit = {},
  timeout = 30000
): Promise<Response> {
  const controller = new AbortController();
  const id = setTimeout(() => controller.abort(), timeout);
  try {
    return await fetch(url, { ...options, signal: controller.signal });
  } finally {
    clearTimeout(id);
  }
}
```

### 2.2 推荐架构改进

#### 2.2.1 API Client 模块化 (TD-008)

**当前**:

**推荐**:

**收益**:
- 更好的代码组织
- 减少单文件复杂度
- 便于 tree-shaking
- IDE 性能提升

#### 2.2.2 后端依赖注入 (TD-005)

**当前**:
```python
# services/openai_service.py
openai_service = OpenAIService()  # 全局单例

# api/chat.py
from services.openai_service import openai_service
```

**推荐**:
```python
# main.py
app.state.openai_service = OpenAIService()
app.state.rag_service = RAGService()

# dependencies.py
def get_openai_service(request: Request) -> OpenAIService:
    return request.app.state.openai_service

# api/chat.py
@router.post("/message")
async def send_message(
    ...,
    openai_service: OpenAIService = Depends(get_openai_service)
):
    ...
```

**收益**:
- 测试时可 mock
- 依赖显式声明
- 更好的可测试性

## 3. 性能优化机会

### 3.1 启动性能

| 优化项 | 当前 | 目标 | 方法 |
|-------|------|------|------|
| 应用启动 | ~2.5s | <1.5s | ChromaDB 懒加载 |
| 首屏渲染 | ~500ms | ~300ms | 组件懒加载 |
| 后端就绪 | ~1s | ~500ms | 移除阻塞初始化 |

**ChromaDB 懒加载方案**:

```python
# services/rag_service.py
class RAGService:
    def __init__(self, ...):
        self._collection = None  # 不立即初始化
        self._chroma_client = None

    async def _ensure_initialized(self):
        """延迟初始化，首次 RAG 使用时执行"""
        if self._collection is None:
            self._chroma_client = AsyncChromaClient(...)
            self._collection = await self._chroma_client.get_or_create_collection(...)
```

### 3.2 内存优化

| 优化项 | 当前 | 目标 | 方法 |
|-------|------|------|------|
| 前端内存 | ~250MB | <200MB | 消息缓存限制 |
| 后端内存 | ~150MB | <150MB | 维持现状 |

**消息缓存限制方案**:

```typescript
// useChat.ts
const MAX_CACHED_MESSAGES = 100;

// 只保留最近 100 条消息在内存
const [messages, setMessages] = useState<Message[]>([]);

const addMessage = (msg: Message) => {
  setMessages(prev => {
    const newMessages = [...prev, msg];
    if (newMessages.length > MAX_CACHED_MESSAGES) {
      // 保留第一条 (系统提示) + 最近 99 条
      return [newMessages[0], ...newMessages.slice(-99)];
    }
    return newMessages;
  });
};
```
