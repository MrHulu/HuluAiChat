## 2. 功能一：DeepSeek 默认模型

### 2.1 技术可行性分析

**结论**: 非常可行，工作量 1 天

DeepSeek 使用 OpenAI 兼容 API，现有的 `OpenAIService` 可以直接复用，只需：

1. 修改默认模型配置
2. 在模型列表中添加 DeepSeek 选项
3. 支持自定义 base_url（DeepSeek API endpoint）

**为什么容易**:
- OpenAI SDK 兼容 DeepSeek API
- 已有多 Provider 支持经验（Ollama）
- 不涉及新的数据结构或 API

## 3. 功能二：RAG 单文档对话

### 3.1 技术可行性分析

**结论**: 可行，复杂度中等，建议分阶段实现

RAG (Retrieval-Augmented Generation) 是成熟技术，Python 生态有丰富的库支持。关键挑战：

1. **文档解析**: PDF 解析需要专门库（PyPDF2 或 unstructured）
2. **向量化**: 需要 embedding 服务（OpenAI 或本地模型）
3. **向量存储**: Chroma 是轻量级嵌入式向量数据库，适合桌面应用
4. **检索质量**: 简单的 chunk 策略可以满足基础需求

## 5. 架构决策记录 (ADR)

### ADR-001: 使用 Chroma 作为向量数据库

**背景**: 需要选择向量存储方案

**选项**:
1. **Chroma** - 嵌入式，Python 原生，适合桌面应用
2. **Pinecone** - 云服务，需要网络连接
3. **FAISS** - Meta 开源，需要更多手动管理
4. **Qdrant** - 功能丰富，但需要独立服务

**决策**: Chroma

**理由**:
- 嵌入式运行，不需要额外服务
- Python 原生，集成简单
- 支持持久化存储
- 社区活跃，文档完善
- 符合"本地优先"的产品理念

**风险**: 大规模数据性能可能不如专用服务，但对于单文档场景足够。

---

### ADR-002: 使用 OpenAI Embedding API

**背景**: 需要选择 embedding 方案

**选项**:
1. **OpenAI text-embedding-3-small** - 高质量，低成本
2. **本地模型 (nomic-embed-text)** - 完全离线，隐私好
3. **Cohere Embed** - 质量好，但需要 API

**决策**: OpenAI 为主，本地模型作为备选

**理由**:
- text-embedding-3-small 价格极低 ($0.02/1M tokens)
- 质量稳定，多语言支持好
- 与 OpenAI LLM 配合效果更好
- 提供本地模型备选保证离线可用

---

### ADR-003: 简单分块策略 (v3.46.0)

**背景**: 需要 chunk 文档以进行检索

**选项**:
1. **固定长度分块** - 简单但可能截断语义
2. **递归字符分块** - 平衡语义和简单性
3. **语义分块** - 复杂，需要额外的 embedding 调用

**决策**: 递归字符分块 (500 字符，100 字符重叠)

**理由**:
- 实现简单，无需额外依赖
- 适合大多数文档类型
- 500 字符是经验值，覆盖大多数段落
- 重叠保证上下文完整性

**后续迭代**: 根据用户反馈优化策略

## 6. 运维考量

### 6.1 存储位置

```
# Windows
%APPDATA%/HuluChat/
  ├── huluchat.db           # SQLite 数据库
  ├── chroma/               # Chroma 向量存储
  └── documents/            # 上传的文档缓存

# macOS
~/Library/Application Support/HuluChat/

# Linux
~/.local/share/HuluChat/
```

### 6.2 资源消耗估算

| 场景 | 内存 | 磁盘 | CPU |
|------|------|------|-----|
| 空闲 | ~100MB | ~50MB | 0% |
| 对话中 | ~200MB | - | 5-10% |
| 处理 5MB 文档 | ~500MB | +15MB | 30-50% |
| Chroma 索引 | ~50MB per doc | +10% | 峰值 |

### 6.3 错误处理策略

1. **文档解析失败**: 返回具体错误，引导用户检查格式
2. **Embedding 失败**: 自动切换到本地模型
3. **Chroma 损坏**: 启动时检测，提示重建索引
4. **内存不足**: 限制并发处理，提示关闭其他文档
