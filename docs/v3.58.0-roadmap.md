# v3.58.0 版本路线图

**版本主题**: 消息交互增强 + 个性化体验 + 技术韧性

**战略目标**: 让用户与 AI 的对话更自然、更高效，同时提升系统稳定性

**开发周期**: 5-6 Cycles（约 2 周）

**审核状态**: ✅ 已通过 Critic Munger 审核（有修改）

---

## 产品分析

### 当前产品状态

**v3.56.0 已完成**:
- AI 知识中心（提示词技巧、模型对比）
- 帮助支持体系（FAQ、反馈入口）
- 新手引导（欢迎流程、功能发现）
- 快捷操作（命令面板、书签跳转）
- 智能提示（上下文提示、错误解决）

**v3.57.0 已完成**:
- 消息重新生成
- 消息编辑
- 会话模板
- 自定义命令
- 批量会话操作

**核心能力**:
- 多模型对话（OpenAI、DeepSeek、Ollama）
- 文档对话（RAG）
- MCP 插件系统
- 会话管理（文件夹、标签、书签）
- 提示词模板、自定义命令

### 用户需求分析

#### 1. 消息交互需求（高频痛点）
- **消息引用**: 回复时引用之前的内容，让 AI 更准确理解上下文
- **消息复制**: 快速复制消息内容（代码、文本）
- **消息搜索**: 在当前会话中搜索关键词

#### 2. 个性化需求（深度用户）
- **主题定制**: 自定义应用外观（亮色/暗色/自定义）
- **快捷键自定义**: 用户自定义快捷键绑定
- **布局配置**: 调整侧边栏、面板布局

#### 3. 导出增强需求（专业用户）
- **多格式导出**: PDF、HTML 导出
- **带样式导出**: 保留消息样式（代码高亮、Markdown 渲染）
- **分享链接**: 生成本地分享文件（不涉及服务器）

### 竞品差异化分析

| 维度 | ChatGPT | Claude Desktop | HuluChat 优势 |
|------|---------|----------------|---------------|
| 隐私 | 数据用于训练 | 免费版训练数据 | **零数据收集** |
| 个性化 | 有限 | 有限 | **高度可定制** |
| 导出 | 基础 | 基础 | **多格式** |
| 引用回复 | ✅ | ✅ | **待实现** |
| 主题 | 固定 | 固定 | **可自定义** |

## 详细功能规格

### TASK-200: 完善引用回复

**描述**: 完善引用回复功能（前端已有引用按钮，需要添加预览和后端支持）

**已有功能**:
- MessageItem 已有 `onQuote` prop 和引用按钮
- 双击消息已支持引用
- 缺少：QuotePreview 组件、后端 API 支持

**需要实现**:
- 前端:
  - QuotePreview 组件（显示在输入框上方）
  - 发送时包含 quoted_message_id
  - 取消引用按钮
- 后端:
  - Chat API 支持 quoted_message_id 参数
  - 将引用消息添加到上下文

**隐私约束**: 引用仅存在于当前会话，不额外存储

**预计**: 0.5 Cycle

### TASK-202: 会话内搜索

**描述**: 用户可以在当前会话中搜索关键词

**用户场景**:
1. 用户在长对话中想找特定内容
2. 按 Ctrl+F 或点击搜索按钮
3. 输入关键词
4. 匹配的消息高亮显示
5. 可跳转到上一个/下一个匹配

**技术实现**:
- 前端:
  - ChatView 添加搜索栏
  - 前端过滤匹配消息
  - 高亮显示匹配文本
  - 滚动到匹配位置

**预计**: 0.5-1 Cycle

---

### TASK-203: 主题定制（仅预设）

**描述**: 用户可以切换预设主题

**主题选项**:
- Light（浅色）
- Dark（深色）
- System（跟随系统）

**注意**: MVP 阶段仅支持预设主题，自定义颜色作为 P2 功能

**技术实现**:
- 前端:
  - SettingsDialog 添加外观 Tab
  - ThemeProvider 支持主题切换
  - localStorage 存储主题偏好

**预计**: 0.5-1 Cycle

---

### TASK-204: 快捷键自定义

**描述**: 用户可以自定义快捷键绑定

**可配置快捷键**:
- 新建会话
- 搜索会话
- 命令面板
- 切换侧边栏
- 发送消息

**技术实现**:
- 前端:
  - SettingsDialog 添加快捷键 Tab
  - 快捷键录制组件
  - 冲突检测
  - localStorage 存储配置

**注意**: 需要先梳理现有硬编码快捷键

**预计**: 1-2 Cycle

---

### TASK-205: 前端健康监控

**描述**: 前端监控后端健康状态，自动恢复

**CTO 建议**: 由于 Rust 编译内存限制，使用前端轮询方案替代 sidecar 健康监控

**功能**:
- 前端定期轮询后端健康状态
- 检测后端崩溃后自动重启
- 显示后端状态指示器
- 重试机制

**技术实现**:
- 前端:
  - useBackendHealth hook
  - 状态指示器组件
  - 自动重启逻辑

**预计**: 1 Cycle

## 关键约束

### 隐私红线

**禁止**:
- 用户行为追踪/埋点
- 向服务器发送使用数据
- 记录用户操作历史

**允许**:
- 本地存储用户配置
- 本地存储主题偏好
- 检测当前状态
