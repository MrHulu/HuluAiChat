# v3.56.0 版本路线图

**版本主题**: AI 知识中心 + 帮助支持体系

**战略目标**: 让每一位用户都能成为 AI 高手，通过知识赋能实现产品价值的最大化

**开发周期**: 9-12 Cycles（约 2-3 周）

---

## 版本概述

v3.56.0 聚焦于用户体验提升，通过知识赋能和帮助体系，让用户更好地使用 AI。本版本不引入新的核心功能，而是优化现有功能的使用体验。

## 开发阶段

### Phase 1: 基础设施（Cycle 1-3）

**目标**: 建立核心框架

#### TASK-183: 命令面板框架
- **描述**: 实现 Ctrl+K 命令面板基础框架
- **内容**:
  - 全局快捷键注册 (Tauri global-shortcut 插件)
  - 命令面板 UI 组件 (fzf 模糊搜索)
  - 命令注册机制
  - 基础导航命令
- **技术栈**: Tauri plugin, fzf, shadcn/ui
- **预计**: 1 Cycle

#### TASK-184: 提示词技巧指南
- **描述**: 创建内置的提示词使用指南
- **内容**:
  - 6-10 个核心技巧文档
  - Markdown 渲染页面
  - 知识中心入口
- **技术栈**: remark + rehype
- **预计**: 1 Cycle

#### TASK-185: FAQ 常见问题
- **描述**: 可折叠 FAQ 列表
- **内容**:
  - 15-20 个常见问题
  - Accordion UI 组件
  - 分类显示
- **技术栈**: shadcn/ui Accordion
- **预计**: 0.5 Cycle

#### TASK-186: 快捷键列表
- **描述**: 显示所有可用快捷键
- **内容**:
  - 快捷键数据收集
  - 表格展示 UI
  - 平台适配 (Cmd/Ctrl)
- **预计**: 0.5 Cycle

### Phase 3: 新手体验（Cycle 6-7）

**目标**: First Impression 优化

#### TASK-190: 首次使用引导
- **描述**: 新用户欢迎引导流程

- **预计**: 1 Cycle

## 技术选型

| 需求 | 选型 | 理由 |
|------|------|------|
| Markdown 解析 | remark + rehype | 生态成熟，已在项目中使用 |
| 前端搜索 | minisearch | 轻量 (~10KB)，功能足够 |
| 模糊搜索 | fzf | 匹配质量好，~15KB |
| 全局快捷键 | tauri-plugin-global-shortcut | Tauri 2.0 官方方案 |
| 折叠组件 | shadcn/ui Accordion | 已在项目中 |

---

## 关键约束

### 隐私红线

**禁止**:
- ❌ 用户行为追踪/埋点
- ❌ 向服务器发送使用数据
- ❌ 本地存储可识别用户身份的信息
- ❌ 记录用户操作历史

**允许**:
- ✅ 检测"当前状态"（如当前页面）
- ✅ 本地存储布尔值（如"是否使用过某功能"）
- ✅ 提供外部链接（如 GitHub Issues）

---

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 内容质量不高 | 用户不信任 | 先做 10 个高质量内容，再扩展 |
| 引导过于繁琐 | 用户跳过 | 控制在 3 步内，可随时跳过 |
| 命令面板性能 | 体验差 | 限制命令数量，优化搜索算法 |
| 功能发现打扰用户 | 负面体验 | 仅在首次出现，可永久关闭 |
| 滑向追踪行为 | 隐私违规 | 严格 Code Review |
