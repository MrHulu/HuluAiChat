# 可分发

## 新增需求

### 需求：可分发

系统应支持将应用打包为独立可执行文件（如 Windows 的 exe），使未安装 Python 运行时的用户也能直接运行应用，且配置与 SQLite 数据仍能正确读写。

#### 场景：Windows 下打包为 exe

- **当** 开发者或 CI 执行打包流程（如使用 PyInstaller 或约定工具）
- **那么** 生成可在目标 Windows 环境运行的 exe（或 exe + 少量依赖文件）
- **并且** 运行 exe 时应用能正常启动、加载配置、访问 SQLite，行为与开发时一致

#### 场景：exe 运行时数据路径

- **当** 用户通过 exe 启动应用
- **那么** 配置文件与 SQLite 数据库存放在用户可预期或文档说明的位置（如用户目录下的应用子目录）
- **并且** 不依赖“exe 所在目录”作为唯一写入位置，避免权限或便携盘导致的问题

#### 场景：依赖与体积

- **当** 打包时
- **那么** 所有运行时依赖（如 CustomTkinter、openai 等）被包含进包内，无需用户额外安装
- **并且** 在满足可运行的前提下，可接受一定体积；若有明确体积约束，可在 design 中约定打包策略（如单文件 vs 目录）
