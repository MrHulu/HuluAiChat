# 多模型接入与切换

## 新增需求

### 需求：多模型接入与切换

系统应支持用户配置多个 AI Provider（OpenAI 兼容接口），并在运行时选择当前用于对话的模型。每个 Provider 包含名称、Base URL、API Key、Model ID 等配置；调用时统一通过同一套接口与所选 Provider 通信。

#### 场景：用户添加并选择 Provider

- **当** 用户在配置中新增一个 Provider（填写 name、base_url、api_key、model_id）
- **那么** 该 Provider 出现在可选模型列表中
- **并且** 用户可将该 Provider 设为当前使用的模型，后续对话使用该模型

#### 场景：用户切换当前模型

- **当** 用户从模型列表中选择另一个已配置的 Provider
- **那么** 当前对话会话（及新消息）使用新选中的模型与 API
- **并且** 切换后无需重启应用即可生效

#### 场景：仅有一个或零个 Provider

- **当** 用户未配置任何 Provider 或仅配置一个
- **那么** 无选择时应用不发起对话或给出明确提示；仅有一个时可直接使用该 Provider
- **并且** 界面不要求用户必须选择（单一 Provider 时可为默认选中）
